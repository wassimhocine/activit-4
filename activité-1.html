Réseau informatique:
est un ensemble d'ordinateurs et de périphériques connectés entre eux, situés dans une certaine zone géographique, qui échangent des informations numériques selon des règles définies.
La mise en réseau:consiste à:
1* relier des ordinateurs pour qu'ils puissent partager des ressources.
2* permettent la communication de données entre applications, quelle que soit la distance qui les sépare.
3* ont évolué pour inclure la voix et la vidéo en plus des données.
Les concepts fondamentaux d'un Réseau : il repose sur deux:
l'interconnexion: qui assure la transmission des données d'un nœud à un autre.
la communication: qui permet l'échange de données entre processus (programmes en cours d'exécution).

Les caractéristiques de base:

* sa topologie (physique et logique)

* le débit (bits/s)

* la distance maximale.

* le nombre maximum de nœuds interconnectés.

Quelques termes couramment utilisés en réseau: la station de travail, le nœud, le serveu

Système de Gestion de Version
Qu’est-ce que Git?

Git est un logiciel de versioning ou de gestion de version créé en 2005 pas le crEateur de Linux.Il permet de conserver un historique des modifications effectuées sur un projet afin de pouvoir rapidement identifier les changements effectués et de revenir à une ancienne version en cas de problème, il facilite grandement la gestion de projets et permet de travailler en équipe de manière beaucoup plus efficace.

A quoi sert concrètement un système de gestion de version ?

pour modifier un site web ou tester une fonctionnalité,il est necessaire d’effectuer une sauvegarde du site avant l’implémentation afin de pouvoir le restaurer en cas de probleme . En ayant plusieurs developpeurs qui travaillent sur le meme site web,L’organisation du travail necessitera :

  1. Que les fonctionnalités sur lesquelles travaille un développeur ne rentrent pas en conflit avec celles d'un autre .
  2. Que chaque développeur sache sur quoi travaillent les autres afin de travailler correctement .
  3. Que chaque développeur possède une version actualisée du site pour tester et implémenter ses fonctionnalités .

c'est pour cela qu'on a créé les logiciels de versioning et Git en particulier.Ils permettent d’effectuer ces opérations mais de manière beaucoup plus robuste et avec de nombreuses fonctionnalités supplémentaires très utiles (possibilité de synchroniser son dossier local avec le serveur, connexion aux autres projets et mise à jour des différences automatisée, etc).Git permet également à différentes versions d’un même fichier de coexister.

Les deux modèles des logiciels de gestion de version : modèle centralisé vs modèle décentralisé

Les logiciels de gestion de version sont aujourd’hui tous construits sur le modèle centralisé basé sur l'hebergement du code de la source du projet sur un serveur central ou le modèle décentralisé/distribué qui a été popularisé par Git , il est basé sur l'hebergement du code source sur un serveur distant et presente des avantages (simplicité/flexibilité du travail, securité ) .

Qu’est-ce que GitHub ?

Git est un logiciel de gestion de version tandis que GitHub est un service en ligne d’hébergement de dépôts Git ( la copie de l’intégralité des fichiers d’un projet et de leur version située sur le serveur central) qui fait office deserveur central pour ces dépôts.

Utiliser Git : ligne de commande, console et interface graphique

pour utiliser et installer git, on opte pour un langage en lignes de commande plutôt qu’une interface graphique pour différentes raisons:
  1. avoir accès à toutes les commandes Git .
  2. Toutes les personnes vont avoir accès aux mêmes commandes .
  3. les interfaces graphiques deviennent faciles a utiliser .

En Windows, pour ouvrir la ligne de commande : Dans la barre de recherche, rechercher cmd

Installation de Git

La façon la plus simple d’installer Git est de télécharger la dernière version sur le site officiel : http://git-scm.com/downloads, d’ouvrir le fichier téléchargé et de suivre les instructions à l’écran en laissant toutes les valeurs par défaut.

Paramétrage de Git

Une fois Git installé, nous allons paramétrer le logiciel afin d’enregistrer certaines données pour ne pas avoir à les fournir à nouveau plus tard. Nous allons notamment ici renseigner un nom d’utilisateur et une adresse mail que Git devra utiliser ensuite. Pour faire cela, nous allons utiliser notre première commande Git qui est la commande git config. Cette commande permet de de voir et modifier les variables de configuration qui contrôlent tous les aspects de l’apparence et du comportement de Git. Nous allons également passer une option --global à notre commande. Les options permettent de personnaliser le comportement par défaut de certaines commandes. .Pour vous assurer que vos informations ont bien été enregistrées, vous pouvez taper git config user.name et git config user.email. Les informations entrées devraient être renvoyées.

Démarrer un dépôt Git

Il existe deux façons de créer un dépôt Git :
  1. On peut importer un répertoire déjà existant dans Git .
  2. On peut cloner un dépôt Git déjà existant.

La gestion des informations selon Git

à chaque fois qu’on va valider ou enregistrer l’état d’un projet dans Git, il va prendre un instantané du contenu de l’espace de travail à ce moment et va enregistrer une référence à cet instantané pour qu’on puisse y accéder par la suite. Chaque instantané est stocké dans une base de données locale, cela permet de realiser localement la grande majorité des opérations de Git.

Les états des fichiers

un fichier peut avoir deux grands états dans Git : il peut être sous suivi de version ,s’il appartenait au dernier instantané capturé par Git, ou non suivi. Lorsqu’on démarre un dépôt Git en important un répertoire déjà existant depuis notre machine, les fichiers sont au départ tous non suivis. On va donc déjà devoir demander à Git de les enregistrer en base( indexer et valider). Lorsqu’on clone un dépôt Git déjà existant, les fichiers sont tous déjà suivis par défaut.
chaque fichier suivi peut avoir l’un de ces trois états :
  1. Modifié (“modified”); le fichier est modifié.
  2. Indexé (“staged”); lorsqu’on indique à Git que le fichier doit faire partie du prochain instantané.
  3. Validé (“committed”). le fichier est stocké dans la base de données.

Les zones de travail

Les états de fichiers sont liés à des zones de travail dans Git. En fonction de son état, un fichier va pouvoir apparaitre dans telle ou telle zone de travail. Tout projet Git est composé de trois sections : le répertoire de travail (working tree), la zone d’index (staging area) et le répertoire Git (repository).

Créer un dépôt Git à partir d’un répertoire existant

pour initialiser un dépôt Git, on utilise la commande git init . Cela crée un sous répertoire .git qui contient un ensemble de fichiers qui vont permettre à un dépôt Git de fonctionner. la commande git status nous permet de verifier si tout les fichier son suivis .Pour indexer des fichiers, on utilise la commande git add pour valider ces fichiers et les ajouter en base, on va maintenant utiliser la commande Git commit.

Cloner un dépôt Git

pour cloner localement un dépôt Git il faut :
  1. utiliser la commande Git clone.
  2. Créer un repo sur GitHub.
  3. transmettre nos fichiers à ce repo:
    -git branch -M main
    -git remote add origin
    -https://github.com/BouchekirRedouane/ExmpleCloneGit.git
    -git push -u origin main

Ajouter ou modifier des fichiers dans un projet et actualiser notre dépôt Git

chaque fois qu’on souhaite enregistrer une modification de fichier ou un ajout de fichier dans le dépôt Git, on va devoir utiliser les commandes git add et git commit c pour enregistrer la dernière version d’un fichier, effectuer un git add juste avant un git commit.

Consulter l’historique des modifications Git

La manière la plus simple de consulter l’historique des modifications Git est d’utiliser la commande git log.

Annuler des modifications apportées à un fichier

On va pouvoir faire cela avec la commande générale (git checkout -- nom-du-fichier) ou la nouvelle commande spécialisée( git restore).

Qu’est ce qu’une branche ?

une branche aussi appelée master en Git est un pointeur vers un commit en particulier. Un commit est un objet qui contient un pointeur vers l’instantané du contenu indexé ainsi que des pointeurs vers le ou les commits le précédant directement. Ainsi, créer une nouvelle branche dans Git crée simplement un nouveau pointeur plutôt que de recopier l’intégralité du répertoire de travail.
